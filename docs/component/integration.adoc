= Vendoring Symfony Bundle Integration
:component-name: vendoring
:package-name: vendoring/vendor

== Purpose

This RC component exposes a Symfony-native bundle facade for host applications.
The component owns its business logic, service definitions, DTO/API surface, persistence mapping, and operational contracts.
The host application should only enable the bundle and import route configuration where Symfony requires host-level route registration.

== Runtime entry points

* Bundle class: `App\Vendoring\VendoringBundle`
* Extension class: `App\Vendoring\DependencyInjection\VendoringExtension`
* Service export: `config/component/services.yaml`
* Route export: `config/component/routes.yaml`

== Host application responsibilities

[source,php]
----
// config/bundles.php
App\Vendoring\VendoringBundle::class => ['all' => true],
----

If the component exports HTTP routes, the host imports the route file using the bundle resource path or the package path agreed for the installation mode.

[source,yaml]
----
# config/routes/vendoring.yaml
vendoring_component:
  resource: '@VendoringBundle/config/component/routes.yaml'
----


== External bundle configuration

The bundle now exposes a canonical `vendoring:` configuration tree for reusable-host consumption.
Hosts should prefer semantic bundle configuration over ad hoc application parameters.

[source,yaml]
----
# config/packages/vendoring.yaml
vendoring:
  secret: '%kernel.secret%'
  observability_dir: '%kernel.project_dir%/var/observability'
  fault_tolerance_dir: '%kernel.project_dir%/var/fault-tolerance'
  feature_flags:
    vendor_portal:
      enabled: true
      cohorts: ['beta']
  alert_thresholds:
    errorLogThreshold: 1
    openBreakerThreshold: 1
    missingProbeThreshold: 1
  rollback_thresholds:
    criticalAlertCodes: ['outbound_circuit_open']
    warningAlertCodes: ['runtime_error_spike', 'probe_artifacts_missing', 'observability_metrics_empty']
----

== Canonical service export

`config/component/services.yaml` is the canonical service graph for bundle consumption.
`config/vendor_services.yaml` remains only as a thin legacy host bridge and must not diverge from the component export.

== Boundary rule

Do not move component business logic into the central App.
The App composes enabled bundles; each RC component remains responsible for its own implementation and export contract.
